# 一、数据目录结构

## 设备路径

```
/storage/emulated/0/Android/data/com.ets100.secondary/files/Download/ETS_secondary/
├── data/       # 试卷索引数据（JSON 文件，包含试卷结构和元数据）
└── resource/   # 答案内容（子文件夹，每个包含 content.json）
```

## resource 子文件夹结构

```
resource/
├── {uuid-1}/
│   ├── content.json    ← 题目内容 + 答案（根据 structure_type 解析）
│   ├── res.json        ← 题型配置（exam_type_list）
│   ├── ctrl.json       ← 答题流程控制
│   ├── info.json       ← 模板渲染数据
│   └── material/       ← 音频、图片素材
├── {uuid-2}/
│   └── content.json
└── ...
```

## data 文件的两种格式

| 格式 | 特征 | 用途 |
|------|------|------|
| **常规习题** | 有 `sectionData` 字段，`newStruct: true` | 正式考试试卷，通过 `fileName` 关联 resource |
| **习题组索引** | 有 `data[]` 字段，`new_struct: 1` | 练习题库索引，答案通过扫描 resource 目录获取 |

# 二、不同地区题型与数据结构

## 地区识别：通过 resource 文件夹数量判断

| 文件夹数 | 地区/学段 | 题型组成 |
|----------|----------|----------|
| **3** | 广东高中 | 3问5答（每题多个参考答案） |
| **7** | 广东初中 | 听选信息 + 回答问题 + 信息转述 + 提问 |
| **10** | 北京初中 | 听后选择 + 听后回答 + 听后转述 |
| **13** | 北京高中 | 听后选择 + 听后记录 + 听后转述 + 回答问题 |

## structure_type 与解析方式

| structure_type | 出现场景 | 答案位置 |
|----------------|----------|----------|
| `collector.role` | 广东初中（听选/回答/提问）、北京初中（听后回答） | `info.question[].std[].value` |
| `collector.3q5a` | 广东高中（3问5答） | `info.question[].std[]`（每题多个答案） |
| `collector.choose` | 北京初中/高中（听后选择） | `info.xtlist[].answer`，选项在 `info.xtlist[].xxlist` |
| `collector.picture` | 各地区（信息转述/听后转述） | `info.std[].value` |
| `collector.fill` | 北京高中（听后记录/填空） | `info.std[].value`，题号在 `info.std[].xth` |
| `collector.dialogue` | 北京高中（回答问题） | `info.question[].std[].value` |
| `collector.read` | 各地区（模仿朗读/短文朗读） | 无答案，跳过 |

## 广东初中题型细分（同为 collector.role 的区分）

广东初中的听选信息、回答问题、提问都使用 `collector.role`，需通过以下条件区分：

| 条件 | 题型 |
|------|------|
| `ask` 包含 `<br>` 且 `askaudio` 有值 | 听选信息 |
| `askaudio` 有值（不含 `<br>`） | 回答问题 |
| `askaudio` 为空且 `ask` 包含中文 | 提问 |

# 三、解析逻辑

## 整体流程

```
Step 1: 扫描 resource/ 目录（优先）
    │  获取所有含 content.json 的子目录
    │  按修改时间降序排序
    │
    ▼
Step 2: 按时间分组
    │  相邻文件夹修改时间间隔 ≤ 3秒 → 同一组
    │
    ▼
Step 3: 根据文件夹数量路由
    │  3 → 广东高中解析器
    │  7 → 广东初中解析器
    │  10 → 北京初中解析器
    │  13 → 北京高中解析器
    │  其他 → 通用解析器
    │
    ▼
Step 4: 遍历每组的 content.json
    │  根据 structure_type 调用对应解析方法
    │  记录已处理的 content.json 路径（去重用）
    │
    ▼
输出结果
```

## Step 2: 时间分组算法

```kotlin
fun groupByTime(folders: List<FolderItem>, thresholdMs: Long = 2000): List<List<FolderItem>> {
    val groups = mutableListOf(mutableListOf(folders[0]))
    for (i in 1 until folders.size) {
        // 使用绝对值判断时间间隔
        if (kotlin.math.abs(folders[i].lastModified - folders[i - 1].lastModified) <= thresholdMs) {
            groups.last().add(folders[i])
        } else {
            groups.add(mutableListOf(folders[i]))
        }
    }
    return groups
}
```

## Step 4: 各地区解析器

### 广东初中（7个文件夹）

遍历每个 content.json：
- `collector.role` → 遍历 `info.question[]`，按 askaudio/中文条件分为听选信息、回答问题、提问
- `collector.picture` → 从 `info.std[].value` 提取信息转述答案
- `collector.read` → 跳过

输出顺序：听选信息 → 回答问题 → 信息转述 → 提问

### 广东高中（3个文件夹）

遍历每个 content.json：
- `collector.3q5a` → 遍历 `info.question[]`，每题从 `std[]` 提取多个参考答案

### 北京初中（10个文件夹）

遍历每个 content.json：
- `collector.choose` → 从 `info.xtlist[]` 提取题目、选项（`xxlist`）、答案（`answer`）
- `collector.role` → 从 `info.question[].std[].value` 提取听后回答
- `collector.picture` → 从 `info.std[].value` 提取听后转述

输出顺序：听后选择 → 听后回答 → 听后转述

### 北京高中（13个文件夹）

遍历每个 content.json：
- `collector.choose` → 听后选择（同北京初中）
- `collector.fill` → 从 `info.std[]` 提取填空答案（`xth` 题号 + `value` 答案）
- `collector.picture` → 听后转述
- `collector.dialogue` → 从 `info.question[].std[].value` 提取回答问题
- `collector.read` → 跳过

输出顺序：听后选择 → 听后记录 → 听后转述 → 回答问题

### 通用解析器（文件夹数量不匹配）

遍历每个 content.json，根据 `structure_type` 分发：
- `collector.role` / `collector.3q5a` / `collector.dialogue` → `info.question[].std[].value`
- `collector.choose` → `info.xtlist[].answer` + `xxlist`
- `collector.fill` → `info.std[].xth` + `value`
- `collector.picture` → `info.std[].value`
- `collector.read` → 跳过

## 文本清洗

解析答案时需清洗文本：
- 移除 `ets_th{数字}` 前缀
- 移除 HTML 标签（`<p>`, `</br>` 等）
- 移除零宽空格（`\u200B`）
- `</p><p>` 替换为换行

# 四、Etsresource 资源索引文件解析

**贡献者**：[hicccc77](https://github.com/hicccc77)

## 目录结构

```
app/src/main/assets/etsresource/
├── beijing-C1.json, beijing-C2.json, beijing-C3.json   ← 北京版初中
├── beijing-G1.json, beijing-G2.json, beijing-G3.json   ← 北京版高中
├── resource-C1.json, resource-C2.json, resource-C3.json ← 通用初中
└── resource-G1.json, resource-G2.json, resource-G3.json ← 通用高中
```

`beijing-*.json` 和 `resource-*.json` 结构相同，仅地区不同。

## beijing-*.json 结构（试卷索引）

```json
{
  "items": [
    {
      "id": "369648",
      "name": "(新教材)BSD 7年级(上) SU1A",
      "fileIdentifiers": ["78d1654b...", "46a34280...", "..."]
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `id` | 试卷唯一标识符 |
| `name` | 试卷名称 |
| `fileIdentifiers` | 关联的 resource 子文件夹标识符数组 |

## resource-*.json 结构（资源映射表）

```json
{
  "byId": {
    "376927": {
      "name": "(新教材)7年级(上)U1A",
      "file_identifiers": ["a1dac348...", "f769c272...", "..."]
    }
  }
}
```

## 数据关联关系

```
beijing-*.json / resource-*.json
    │
    ├── items[].id / byId.keys → paperId
    ├── items[].fileIdentifiers / byId[].file_identifiers → resource 子文件夹名
    │
    ▼
data/{paperId} → sectionData → fileName → resource/{fileName}/content.json
```

## 地区差异

| 系列 | 命名格式 | fileIdentifiers 数量 |
|------|----------|---------------------|
| C系列（初中） | `{年级}(上/下){单元}` | 约 7 个 |
| G系列（高中） | `{必修/选择性必修}{数字} {单元}` | 约 3 个（通用）/ 12-13 个（北京） |
